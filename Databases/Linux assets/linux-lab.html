Linux Automation Lab

A detailed walkthrough of the Bash scripting challenge, covering file creation automation, permission management, and output sorting.

1 The Assignment Brief

The objective was to create a script that generates 25 distinct text files named sequentially (e.g., Neo1.txt, Neo2.txt...). This tests the ability to use loops and variables within a Bash shell environment.

  • Connect via SSH to EC2 Instance
  • Create working directory
  • Initialize script file

2 Writing the Script

Using the nano editor, I wrote a simple for loop. The script iterates from 1 to 25, appending the iterator number to my name variable to create unique filenames.

chmod +x create_files.sh

Don't forget to make the script executable before running it!

3 Running the Code

Upon execution, the files were created successfully. However, listing them with a standard ls -l command revealed a sorting issue. Linux sorts lexicographically by default, meaning "10" comes right after "1".

4 Fixing the Output

To verify the correct creation order, I used the ls -lv command (natural sort). This displayed the files in the intuitive numerical order (1, 2, 3... 10) rather than the default ASCII sort.

Success! All 25 files verified and sorted correctly.
